# Security policy

## Supported versions

We release patches for security issues for the current major version. Check the [releases](https://github.com/your-org/llmfy/releases) page for supported versions.

## Reporting a vulnerability

If you believe you have found a security vulnerability, please report it responsibly:

1. **Do not** open a public issue.
2. Email the maintainers (or open a **private** security advisory on GitHub if the repo supports it) with:
   - A short description of the issue
   - Steps to reproduce
   - Impact (e.g. local only, data exposure)
   - Any suggested fix, if you have one

We will acknowledge your report and work on a fix. We may ask for more details. Once a fix is ready, we will release a patch and credit you in the release notes (unless you prefer to stay anonymous).

## What’s in scope

- The llmfy CLI and its dependencies (including hardware detection, benchmark, and Ollama install flow).
- Out-of-scope: vulnerabilities in Ollama, Node.js, or the OS; please report those to the respective projects.

Thank you for helping keep llmfy safe.
